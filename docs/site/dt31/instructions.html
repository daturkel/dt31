  10class Instruction:
  11    """Base class for all DT31 instructions.
  12
  13    Instructions are the fundamental building blocks of DT31 programs. Each instruction
  14    performs a specific operation when executed by the CPU, such as arithmetic, logic,
  15    memory manipulation, or control flow.
  16
  17    Execution Model
  18    ---------------
  19    When an instruction is executed (via `__call__`), it follows a two-phase process:
  20
  21    1. **Calculation phase** (`_calc`): Computes the instruction's result value and
  22       performs any side effects (e.g., writing to memory, pushing to stack).
  23    2. **Advance phase** (`_advance`): Updates the instruction pointer (IP) to determine
  24       which instruction executes next. By default, increments IP by 1, but jump
  25       instructions override this to change control flow.
  26
  27    Implementing New Instructions
  28    ------------------------------
  29    To create a new instruction, subclass `Instruction` and implement:
  30
  31    - `_calc(cpu)`: Perform the instruction's operation and return a result value.
  32      This value is available to the instruction but typically only used for operations
  33      that need to store results (via `BinaryOperation` or `UnaryOperation` base classes).
  34
  35    - `_advance(cpu)` (optional): Override to customize how the instruction pointer moves.
  36      Default behavior increments IP by 1. Jump instructions override this to modify
  37      control flow.
  38
  39    - `__str__()` (optional): Return a human-readable representation showing the
  40      instruction name and operands for debugging and display purposes.
  41
  42    Examples
  43    --------
  44    Simple instruction (no operands, no special `_advance` behavior):
  45    ```python
  46    class NOOP(Instruction):
  47        def __init__(self):
  48            super().__init__("NOOP")
  49
  50        def _calc(self, cpu: DT31) -> int:
  51            return 0  # Do nothing
  52    ```
  53
  54    Jump instruction (custom `_advance`, `__str__`):
  55    ```python
  56    class JMP(Instruction):
  57        def __init__(self, dest: Operand | int):
  58            super().__init__("JMP")
  59            self.dest = as_op(dest)
  60
  61        def _calc(self, cpu: DT31) -> int:
  62            return 0
  63
  64        def _advance(self, cpu: DT31):
  65            cpu.set_register("ip", self.dest.resolve(cpu))
  66
  67        def __str__(self) -> str:
  68            return f"{self.name}(dest={self.dest})"
  69    ```
  70    """
  71
  72    def __init__(self, name: str):
  73        """Initialize an Instruction.
  74
  75        Args:
  76            name: The name of the instruction (e.g., "ADD", "JMP", "PUSH").
  77        """
  78        self.name = name
  79
  80    def _calc(self, cpu: DT31) -> int:
  81        """Perform the instruction's operation and return a result value.
  82
  83        This method must be implemented by subclasses to define the instruction's
  84        behavior. It performs the core operation (arithmetic, comparison, I/O, etc.)
  85        and returns an integer result.
  86
  87        Args:
  88            cpu: The DT31 CPU instance executing this instruction.
  89
  90        Returns:
  91            The computed result value. For instructions that write to operands
  92            (via `BinaryOperation` or `UnaryOperation`), this value is stored
  93            in the output location. For other instructions, the return value
  94            may be unused.
  95
  96        Raises:
  97            NotImplementedError: Must be implemented by subclasses.
  98        """
  99        raise NotImplementedError()
 100
 101    def _advance(self, cpu: DT31):
 102        """Update the instruction pointer to determine the next instruction.
 103
 104        By default, increments the instruction pointer by 1 to execute the next
 105        sequential instruction. Jump instructions override this method to modify
 106        control flow (e.g., jumping to a different location or returning from a call).
 107
 108        Args:
 109            cpu: The DT31 CPU instance executing this instruction.
 110        """
 111        # default behavior is to increment the instruction register by 1
 112        cpu.set_register("ip", cpu.get_register("ip") + 1)
 113
 114    def __call__(self, cpu: DT31) -> int:
 115        """Execute the instruction on the given CPU.
 116
 117        This is the main entry point for instruction execution. It orchestrates
 118        the two-phase execution model by calling `_calc` followed by `_advance`.
 119
 120        This function is overridden in some of the subclasses like `UnaryOperation` and
 121        `BinaryOperation` to handle storing results.
 122
 123        Args:
 124            cpu: The DT31 CPU instance to execute this instruction on.
 125
 126        Returns:
 127            The result value computed by `_calc`.
 128        """
 129        value = self._calc(cpu)
 130        self._advance(cpu)
 131        return value
 132
 133    def __str__(self) -> str:
 134        """Return a string representation of the instruction.
 135
 136        Returns:
 137            A string showing the instruction name and any operands for debugging.
 138        """
 139        return f"{self.name}()"

 152class UnaryOperation(Instruction):
 153    """Base class for instructions which modify a single operand and optionally write
 154    to a separate operand."""
 155
 156    def __init__(self, name: str, a: Operand | int, out: Reference | None = None):
 157        super().__init__(name)
 158        self.a = as_op(a)
 159        if not isinstance(out, (type(None), Reference)):
 160            raise ValueError("argument `out` must be a Reference or None")
 161        if out is not None:
 162            self.out = out
 163        elif isinstance(self.a, Reference):
 164            self.out = self.a
 165        else:
 166            raise ValueError(
 167                f"{self.name} must be called with a reference as operand `out` or a reference as operand `a`"
 168            )
 169
 170    def __call__(self, cpu: DT31) -> int:
 171        value = super().__call__(cpu)
 172        cpu[self.out] = value
 173        return value
 174
 175    def __str__(self) -> str:
 176        return f"{self.name}(a={self.a}, out={self.out})"

 624        raise NotImplementedError()

 627        raise NotImplementedError()

 630        return 0

 633        if self._jump_condition(cpu):

Base class for all DT31 instructions.

Instructions are the fundamental building blocks of DT31 programs. Each instruction performs a specific operation when executed by the CPU, such as arithmetic, logic, memory manipulation, or control flow.

Execution Model

When an instruction is executed (via __call__), it follows a two-phase process:

  1. Calculation phase (_calc): Computes the instruction's result value and performs any side effects (e.g., writing to memory, pushing to stack).
  2. Advance phase (_advance): Updates the instruction pointer (IP) to determine which instruction executes next. By default, increments IP by 1, but jump instructions override this to change control flow.

Implementing New Instructions

To create a new instruction, subclass Instruction and implement:

  • _calc(cpu): Perform the instruction's operation and return a result value. This value is available to the instruction but typically only used for operations that need to store results (via BinaryOperation or UnaryOperation base classes).

  • _advance(cpu) (optional): Override to customize how the instruction pointer moves. Default behavior increments IP by 1. Jump instructions override this to modify control flow.

  • __str__() (optional): Return a human-readable representation showing the instruction name and operands for debugging and display purposes.

Base class for all DT31 instructions.

Instructions are the fundamental building blocks of DT31 programs. Each instruction performs a specific operation when executed by the CPU, such as arithmetic, logic, memory manipulation, or control flow.

Execution Model

When an instruction is executed (via __call__), it follows a two-phase process:

  1. Calculation phase (_calc): Computes the instruction's result value and performs any side effects (e.g., writing to memory, pushing to stack).
  2. Advance phase (_advance): Updates the instruction pointer (IP) to determine which instruction executes next. By default, increments IP by 1, but jump instructions override this to change control flow.

Implementing New Instructions

To create a new instruction, subclass Instruction and implement:

  • _calc(cpu): Perform the instruction's operation and return a result value. This value is available to the instruction but typically only used for operations that need to store results (via BinaryOperation or UnaryOperation base classes).

  • _advance(cpu) (optional): Override to customize how the instruction pointer moves. Default behavior increments IP by 1. Jump instructions override this to modify control flow.

  • __str__() (optional): Return a human-readable representation showing the instruction name and operands for debugging and display purposes.

Base class for all DT31 instructions.

Instructions are the fundamental building blocks of DT31 programs. Each instruction performs a specific operation when executed by the CPU, such as arithmetic, logic, memory manipulation, or control flow.

Execution Model

When an instruction is executed (via __call__), it follows a two-phase process:

  1. Calculation phase (_calc): Computes the instruction's result value and performs any side effects (e.g., writing to memory, pushing to stack).
  2. Advance phase (_advance): Updates the instruction pointer (IP) to determine which instruction executes next. By default, increments IP by 1, but jump instructions override this to change control flow.

Implementing New Instructions

To create a new instruction, subclass Instruction and implement:

  • _calc(cpu): Perform the instruction's operation and return a result value. This value is available to the instruction but typically only used for operations that need to store results (via BinaryOperation or UnaryOperation base classes).

  • _advance(cpu) (optional): Override to customize how the instruction pointer moves. Default behavior increments IP by 1. Jump instructions override this to modify control flow.

  • __str__() (optional): Return a human-readable representation showing the instruction name and operands for debugging and display purposes.
