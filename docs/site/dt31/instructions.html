  14class Instruction:
  15    """Base class for all DT31 instructions.
  16
  17    Instructions are the fundamental building blocks of DT31 programs. Each instruction
  18    performs a specific operation when executed by the CPU, such as arithmetic, logic,
  19    memory manipulation, or control flow.
  20
  21    Execution Model
  22    ---------------
  23    When an instruction is executed (via `__call__`), it follows a two-phase process:
  24
  25    1. **Calculation phase** (`_calc`): Computes the instruction's result value and
  26       performs any side effects (e.g., writing to memory, pushing to stack).
  27    2. **Advance phase** (`_advance`): Updates the instruction pointer (IP) to determine
  28       which instruction executes next. By default, increments IP by 1, but jump
  29       instructions override this to change control flow.
  30
  31    Implementing New Instructions
  32    ------------------------------
  33    To create a new instruction, subclass `Instruction` (or, even better, a helper subclass
  34    like `NullaryOperation`, `UnaryOperation`, or `BinaryOperation`) and implement:
  35
  36    - `__init__(self)`: Pass the instruction's args to the `__init__` method of the parent
  37      class.
  38    - `_calc(cpu)`: Perform the instruction's operation and return a result value.
  39      This value is available to the instruction but typically only used for operations
  40      that need to store results (via `NullaryOperation`,`BinaryOperation` or `UnaryOperation`
  41      base classes).
  42    - `_advance(cpu)` (optional): Override to customize how the instruction pointer moves.
  43      Default behavior increments IP by 1. Jump instructions override this to modify
  44      control flow.
  45    - `__str__()` (optional): Return a human-readable representation showing the
  46      instruction name and operands for debugging and display purposes.
  47
  48    Consult the source of existing instructions as a guide.
  49
  50    Examples
  51    --------
  52    Simple unary operation (square a value):
  53    ```python
  54    class SQUARE(UnaryOperation):
  55        \"\"\"Square a value (a * a).\"\"\"
  56        def __init__(self, a: Operand, out: Reference | None = None):
  57            super().__init__("SQUARE", a, out)
  58
  59        def _calc(self, cpu: DT31) -> int:
  60            val = self.a.resolve(cpu)
  61            return val * val
  62    ```
  63
  64    Custom instruction with multiple operands (swap two registers):
  65    ```python
  66    class SWAP(Instruction):
  67        \"\"\"Swap the values of two registers.\"\"\"
  68        def __init__(self, a: Reference, b: Reference):
  69            super().__init__("SWAP")
  70            self.a = a
  71            self.b = b
  72
  73        def _calc(self, cpu: DT31) -> int:
  74            val_a = self.a.resolve(cpu)
  75            val_b = self.b.resolve(cpu)
  76            self.a.assign(cpu, val_b)
  77            self.b.assign(cpu, val_a)
  78            return 0
  79
  80        def __str__(self) -> str:
  81            return f"{self.name}({self.a}, {self.b})"
  82    ```
  83    """
  84
  85    def __init__(self, name: str):
  86        """Initialize an Instruction.
  87
  88        Args:
  89            name: The name of the instruction (e.g., "ADD", "JMP", "PUSH").
  90        """
  91        self.name = name
  92
  93    def _calc(self, cpu: DT31) -> int:
  94        """Perform the instruction's operation and return a result value.
  95
  96        This method must be implemented by subclasses to define the instruction's
  97        behavior. It performs the core operation (arithmetic, comparison, I/O, etc.)
  98        and returns an integer result.
  99
 100        Args:
 101            cpu: The DT31 CPU instance executing this instruction.
 102
 103        Returns:
 104            The computed result value. For instructions that write to operands
 105            (via `BinaryOperation` or `UnaryOperation`), this value is stored
 106            in the output location. For other instructions, the return value
 107            may be unused.
 108
 109        Raises:
 110            NotImplementedError: Must be implemented by subclasses.
 111        """
 112        raise NotImplementedError()
 113
 114    def _advance(self, cpu: DT31):
 115        """Update the instruction pointer to determine the next instruction.
 116
 117        By default, increments the instruction pointer by 1 to execute the next
 118        sequential instruction. Jump instructions override this method to modify
 119        control flow (e.g., jumping to a different location or returning from a call).
 120
 121        Args:
 122            cpu: The DT31 CPU instance executing this instruction.
 123        """
 124        # default behavior is to increment the instruction register by 1
 125        cpu.set_register("ip", cpu.get_register("ip") + 1)
 126
 127    def __call__(self, cpu: DT31) -> int:
 128        """Execute the instruction on the given CPU.
 129
 130        This is the main entry point for instruction execution. It orchestrates
 131        the two-phase execution model by calling `_calc` followed by `_advance`.
 132
 133        This function is overridden in some of the subclasses like `UnaryOperation` and
 134        `BinaryOperation` to handle storing results.
 135
 136        Args:
 137            cpu: The DT31 CPU instance to execute this instruction on.
 138
 139        Returns:
 140            The result value computed by `_calc`.
 141        """
 142        value = self._calc(cpu)
 143        self._advance(cpu)
 144        return value
 145
 146    def __repr__(self) -> str:
 147        """Return Python API representation of the instruction.
 148
 149        Returns:
 150            A string showing the instruction name and any operands in Python syntax.
 151        """
 152        return f"{self.name}()"
 153
 154    def __str__(self) -> str:
 155        """Return assembly text representation of the instruction.
 156
 157        Returns:
 158            A string showing the instruction in assembly text format.
 159        """
 160        return self.name
 161
 162    def __eq__(self, other):
 163        if type(self) is not type(other):
 164            return False
 165        return self.__dict__ == other.__dict__

 178class NullaryOperation(Instruction):
 179    """Base class for instructions which take no input operands and write to an output
 180    operand."""
 181
 182    out: Reference  # Always set to a Reference in __init__
 183
 184    def __init__(self, name: str, out: Reference):
 185        """Base class for instructions which take no input operands and write to an
 186        output operand.
 187
 188        Args:
 189            name: The name of the instruction (e.g., "RND").
 190            out: Output reference for result.
 191        """
 192        super().__init__(name)
 193        if not isinstance(out, Reference):
 194            raise ValueError("argument `out` must be a Reference")
 195        self.out = out
 196

 198        value = super().__call__(cpu)
 199        cpu[self.out] = value
 200        return value

 204        return f"{self.name}(out={self.out!r})"

Base class for all DT31 instructions.

Instructions are the fundamental building blocks of DT31 programs. Each instruction performs a specific operation when executed by the CPU, such as arithmetic, logic, memory manipulation, or control flow.

Execution Model

When an instruction is executed (via __call__), it follows a two-phase process:

  1. Calculation phase (_calc): Computes the instruction's result value and performs any side effects (e.g., writing to memory, pushing to stack).
  2. Advance phase (_advance): Updates the instruction pointer (IP) to determine which instruction executes next. By default, increments IP by 1, but jump instructions override this to change control flow.

Implementing New Instructions

To create a new instruction, subclass Instruction (or, even better, a helper subclass like NullaryOperation, UnaryOperation, or BinaryOperation) and implement:

  • __init__(self): Pass the instruction's args to the __init__ method of the parent class.
  • _calc(cpu): Perform the instruction's operation and return a result value. This value is available to the instruction but typically only used for operations that need to store results (via NullaryOperation,BinaryOperation or UnaryOperation base classes).
  • _advance(cpu) (optional): Override to customize how the instruction pointer moves. Default behavior increments IP by 1. Jump instructions override this to modify control flow.
  • __str__() (optional): Return a human-readable representation showing the instruction name and operands for debugging and display purposes.

Consult the source of existing instructions as a guide.
