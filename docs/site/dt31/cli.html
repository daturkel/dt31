dt31.cli

Command-line interface for dt31.

This module provides the dt31 command-line executable for parsing, executing, and formatting dt31 assembly files. The CLI automatically detects registers used in programs and validates syntax before execution.

Installation

After installing the dt31 package, the dt31 command becomes available:

pip install dt31

Commands

The dt31 CLI provides three main commands:

  • run: Execute .dt assembly files
  • check: Validate syntax of .dt assembly files
  • format: Format .dt assembly files with consistent style

Basic Usage

dt31 run program.dt       # Execute program
dt31 check program.dt     # Validate syntax
dt31 format program.dt    # Format file in-place

Run Command

Execute .dt assembly files with configurable CPU settings.

Options:

  • file (required): Path to the .dt assembly file to execute
  • -d, --debug: Enable step-by-step debug output during execution
  • -i, --custom-instructions: Path to Python file containing custom instruction definitions
  • --registers: Comma-separated list of register names (auto-detected by default)
  • --memory: Memory size in bytes (default: 256)
  • --stack-size: Stack size (default: 256)
  • --dump: When to dump CPU state - 'none' (default), 'error', 'success', or 'all'
  • --dump-file: File path for dump (auto-generates timestamped filename if not specified)

Examples:

# Execute a program
dt31 run countdown.dt

# Run with debug output
dt31 run --debug program.dt

# Use custom memory size
dt31 run --memory 1024 program.dt

# Specify registers explicitly
dt31 run --registers a,b,c,d,e program.dt

# Load custom instructions
dt31 run --custom-instructions my_instructions.py program.dt

# Dump CPU state on error
dt31 run --dump error --dump-file crash.json program.dt
dt31 run --dump error program.dt  # Auto-generates program_crash_TIMESTAMP.json

# Dump CPU state after successful execution
dt31 run --dump success --dump-file final.json program.dt
dt31 run --dump success program.dt  # Auto-generates program_final_TIMESTAMP.json

# Dump on both error and success
dt31 run --dump all program.dt  # Auto-generates timestamped files

Check Command

Validate the syntax of .dt assembly files without executing them.

Options:

  • file (required): Path to the .dt assembly file to validate
  • -i, --custom-instructions: Path to Python file containing custom instruction definitions

Examples:

# Validate syntax of a program
dt31 check program.dt

# Validate with custom instructions
dt31 check --custom-instructions custom.py program.dt

Exit Codes (Check Command):

  • 0: File is valid
  • 1: Error (file not found, parse error, or custom instruction error)

Format Command

Format .dt assembly files with consistent style, following Black/Ruff conventions (formats in-place by default).

Options:

  • file (required): Path to the .dt assembly file to format
  • --check: Check if formatting is needed without modifying the file (exit 1 if changes needed)
  • --diff: Show formatting changes as a unified diff without modifying the file
  • -i, --custom-instructions: Path to Python file containing custom instruction definitions
  • --indent-size: Number of spaces per indentation level (default: 4)
  • --label-inline: Place labels on same line as next instruction (default: False)
  • --blank-lines: Control blank line handling: 'preserve' (default), 'auto', or 'none'
  • --align-comments: Align inline comments (auto-calculates column if --comment-column not specified)
  • --comment-column: Column position for aligned comments (default: auto-calculate)
  • --comment-margin: Spaces before inline comments and margin for auto-alignment (default: 2)
  • --strip-comments: Remove all comments from output (default: False)
  • --show-default-args: Show instruction arguments even when they match defaults (default: False)

Examples:

# Format file in-place
dt31 format program.dt

# Check if formatting is needed (CI/pre-commit)
dt31 format --check program.dt

# Preview formatting changes
dt31 format --diff program.dt

# Format with custom style
dt31 format --indent-size 2 --label-inline program.dt

# Show default arguments
dt31 format --show-default-args program.dt

# Auto-align comments (calculates optimal column)
dt31 format --align-comments program.dt

# Align comments at specific column
dt31 format --align-comments --comment-column 40 program.dt

# Auto-align with custom margin
dt31 format --align-comments --comment-margin 4 program.dt

# Strip all comments from output
dt31 format --strip-comments program.dt

# Format file with custom instructions
dt31 format --custom-instructions my_instructions.py program.dt

# Check and show diff if needed
dt31 format --check --diff program.dt

Exit Codes (Format Command):

  • 0: Success (formatted, already formatted, or --check passed)
  • 1: Error (file not found, parse error, --check failed, IO error)

Register Auto-Detection

The CLI automatically detects which registers are used in your program and creates a CPU with exactly those registers. This eliminates the need to manually specify registers in most cases.

If you explicitly provide --registers, the CLI validates that all registers used in the program are included in your list.

Exit Codes

  • 0: Success
  • 1: Error (file not found, parse error, runtime error, or CPU creation error)
  • 130: Execution interrupted (Ctrl+C)

Error Handling

The CLI provides helpful error messages for common issues:

  • File not found: Clear message indicating which file couldn't be found
  • Parse errors: Line number and description of syntax errors
  • Runtime errors: Exception message with optional CPU state (in debug mode)
  • Register errors: List of missing registers when validation fails

CPU State Dumps

The --dump option saves complete CPU state to JSON for debugging:

  • Error dumps (--dump error or --dump all): Include CPU state, error info, traceback, and the failing instruction in both repr and str formats
  • Success dumps (--dump success or --dump all): Include final CPU state after successful execution

Dumps are auto-saved with timestamped filenames (program_crash_YYYYMMDD_HHMMSS.json or program_final_YYYYMMDD_HHMMSS.json) unless --dump-file specifies a custom path.

Example error dump structure:

{
  "cpu_state": {
    "registers": {...},
    "memory": [...],
    "stack": [...],
    "program": "...",
    "config": {...}
  },
  "error": {
    "type": "ZeroDivisionError",
    "message": "integer division or modulo by zero",
    "instruction": {
      "repr": "DIV(a=R.a, b=R.b, out=R.a)",
      "str": "DIV R.a, R.b, R.a"
    },
    "traceback": "..."
  }
}

 238def expand_file_patterns(patterns: list[str]) -> list[str]:
 239    """Expand glob patterns to actual file paths.
 240
 241    Args:
 242        patterns: List of file paths and/or glob patterns
 243
 244    Returns:
 245        List of resolved file paths (sorted)
 246
 247    Example:
 248        >>> expand_file_patterns(["program.dt", "*.dt"])
 249        ['program.dt', 'hello.dt', 'countdown.dt']
 250    """
 251    expanded_files = []
 252    for pattern in patterns:
 253        # Check if pattern contains glob characters
 254        if any(char in pattern for char in ["*", "?", "[", "]"]):
 255            # Use glob to expand the pattern
 256            matches = glob.glob(pattern, recursive=True)
 257            if matches:
 258                expanded_files.extend(matches)
 259            # If no matches, keep the original pattern (will error later)
 260        else:
 261            # Not a glob pattern, add as-is
 262            expanded_files.append(pattern)
 263
 264    # Remove duplicates and sort
 265    return sorted(set(expanded_files))

 382def run_command(args: argparse.Namespace) -> None:
 383    """Execute the 'run' subcommand - parse and execute a dt31 program.
 384
 385    Args:
 386        args: Parsed command-line arguments from argparse
 387
 388    This function implements the complete execution workflow:
 389    1. Load custom instructions (if provided)
 390    2. Read and parse the assembly file
 391    3. Auto-detect registers used in the program
 392    4. Create CPU with appropriate configuration
 393    5. Execute the program with optional debug output
 394    6. Handle errors and dump CPU state if requested
 395
 396    Exit codes:
 397        0: Program executed successfully or passed validation (--parse-only)
 398        1: Error occurred (file not found, parse error, runtime error, etc.)
 399        130: User interrupted execution (Ctrl+C)
 400    """
 401    # Load custom instructions if provided
 402    custom_instructions = None
 403    if args.custom_instructions:
 404        try:
 405            custom_instructions = load_custom_instructions(args.custom_instructions)
 406            if args.debug:
 407                print(
 408                    f"Loaded {len(custom_instructions)} custom instruction(s): "
 409                    f"{', '.join(custom_instructions.keys())}",
 410                    file=sys.stderr,
 411                )
 412        except (FileNotFoundError, ImportError, ValueError, TypeError) as e:
 413            print(f"Error loading custom instructions: {e}", file=sys.stderr)
 414            sys.exit(1)
 415
 416    # Read the input file
 417    file_path = Path(args.file)
 418    try:
 419        assembly_text = file_path.read_text()
 420    except FileNotFoundError:
 421        print(f"Error: File not found: {args.file}", file=sys.stderr)
 422        sys.exit(1)
 423    except IOError as e:
 424        print(f"Error reading file {args.file}: {e}", file=sys.stderr)
 425        sys.exit(1)
 426
 427    # Parse the assembly program with custom instructions
 428    try:
 429        program = parse_program(assembly_text, custom_instructions=custom_instructions)
 430    except ParserError as e:
 431        print(f"Parse error: {e}", file=sys.stderr)
 432        sys.exit(1)
 433
 434    # Extract registers used in the program
 435    registers_used = extract_registers_from_program(program)
 436
 437    # Create CPU with custom configuration
 438    cpu_kwargs = {}
 439    if args.memory is not None:
 440        cpu_kwargs["memory_size"] = args.memory
 441    if args.stack_size is not None:
 442        cpu_kwargs["stack_size"] = args.stack_size
 443
 444    if args.registers:
 445        # User provided explicit registers - validate they include all used registers
 446        user_registers = args.registers.split(",")
 447        missing = set(registers_used) - set(user_registers)
 448        if missing:
 449            print(
 450                f"Error: Program uses registers {registers_used} but --registers only specified {user_registers}",
 451                file=sys.stderr,
 452            )
 453            print(f"Missing registers: {sorted(missing)}", file=sys.stderr)
 454            sys.exit(1)
 455        cpu_kwargs["registers"] = user_registers
 456    elif registers_used:
 457        # Auto-detect registers from program
 458        cpu_kwargs["registers"] = registers_used
 459    # else: no registers used, CPU will use defaults
 460
 461    try:
 462        cpu = DT31(**cpu_kwargs)
 463    except Exception as e:
 464        print(f"Error creating CPU: {e}", file=sys.stderr)
 465        sys.exit(1)
 466
 467    # Execute the program
 468    try:
 469        cpu.run(program, debug=args.debug)
 470    except (EOFError, KeyboardInterrupt):
 471        # Handle interrupt gracefully (e.g., Ctrl+C during debug mode input)
 472        print("\n\nExecution interrupted", file=sys.stderr)
 473        sys.exit(130)
 474    except Exception as e:
 475        print(f"\nRuntime error: {e}", file=sys.stderr)
 476        if args.debug:
 477            state = cpu.state
 478            print("\nCPU state at error:", file=sys.stderr)
 479            # Print registers (keys starting with R.)
 480            registers = {k: v for k, v in state.items() if k.startswith("R.")}
 481            print(f"  Registers: {registers}", file=sys.stderr)
 482            print(f"  Stack size: {len(state['stack'])}", file=sys.stderr)
 483
 484        # Dump CPU state to file if requested
 485        if args.dump in ("error", "all"):
 486            dump_path = generate_dump_path(args.file, args.dump_file, "crash")
 487            try:
 488                dump_cpu_state(cpu, dump_path, error=e)
 489                print(f"CPU state dumped to: {dump_path}", file=sys.stderr)
 490            except Exception as dump_error:
 491                print(f"Failed to dump CPU state: {dump_error}", file=sys.stderr)
 492
 493        sys.exit(1)
 494
 495    # Dump CPU state on exit if requested
 496    if args.dump in ("success", "all"):
 497        dump_path = generate_dump_path(args.file, args.dump_file, "final")
 498        try:
 499            dump_cpu_state(cpu, dump_path)
 500            print(f"CPU state dumped to: {dump_path}", file=sys.stderr)
 501        except Exception as dump_error:
 502            print(f"Failed to dump CPU state: {dump_error}", file=sys.stderr)
 503
 504    # Success
 505    sys.exit(0)

 508def check_command(args: argparse.Namespace) -> None:
 509    """Execute the 'check' subcommand - validate syntax of a dt31 program.
 510
 511    Args:
 512        args: Parsed command-line arguments from argparse
 513
 514    This function implements the syntax validation workflow:
 515    1. Load custom instructions (if provided)
 516    2. Expand glob patterns to file paths
 517    3. Read and parse each assembly file
 518    4. Report success or error for each file
 519
 520    Exit codes:
 521        0: All files are valid
 522        1: Error occurred (file not found, parse error, etc.)
 523    """
 524    # Load custom instructions if provided
 525    custom_instructions = None
 526    if args.custom_instructions:
 527        try:
 528            custom_instructions = load_custom_instructions(args.custom_instructions)
 529        except (FileNotFoundError, ImportError, ValueError, TypeError) as e:
 530            print(f"Error loading custom instructions: {e}", file=sys.stderr)
 531            sys.exit(1)
 532
 533    # Expand glob patterns
 534    file_paths = expand_file_patterns(args.files)
 535
 536    if not file_paths:
 537        print("Error: No files matched the provided patterns", file=sys.stderr)
 538        sys.exit(1)
 539
 540    # Track results
 541    failed_files = []
 542
 543    # Process each file
 544    for file_path_str in file_paths:
 545        file_path = Path(file_path_str)
 546        # Use relative path for display if possible, otherwise use the original path
 547        try:

 715    file_path: str,
 716    custom_instructions: dict[str, type[Instruction]] | None,
 717    check_mode: bool,
 718    show_diff: bool,
 719    formatting_options: dict,
 720) -> bool:
 721    """Format a single dt31 assembly file.
 722
 723    This is a helper function that handles formatting one file. It's separated
 724    to make it easy to add glob support in the future by wrapping this in a loop.
 725
 726    Args:
 727        file_path: Path to the file to format
 728        custom_instructions: Optional custom instructions dict
 729        check_mode: If True, don't modify file, just check if changes needed
 730        show_diff: If True, don't modify file, just show unified diff of changes
 731        formatting_options: Dict of formatting options to pass to program_to_text()
 732
 733    Returns:
 734        True if file needs formatting (or was formatted), False if already formatted
 735
 736    Raises:
 737        SystemExit: On file not found, parse error, or IO error
 738    """
 739    path = Path(file_path)
 740
 741    # Read the input file
 742    try:
 743        original_text = path.read_text()
 744    except FileNotFoundError:
 745        print(f"Error: File not found: {file_path}", file=sys.stderr)
 746        sys.exit(1)
 747    except IOError as e:
 748        print(f"Error reading file {file_path}: {e}", file=sys.stderr)
 749        sys.exit(1)
 750
 751    # Parse the program
 752    # Extract blank_lines from formatting_options to determine if we need to preserve newlines during parsing
 753    blank_lines = formatting_options.get("blank_lines", "preserve")
 754    preserve_newlines = blank_lines == "preserve"
 755    try:
 756        program = parse_program(
 757            original_text,
 758            custom_instructions=custom_instructions,
 759            preserve_newlines=preserve_newlines,
 760        )
 761    except ParserError as e:
 762        print(f"Parse error: {e}", file=sys.stderr)
 763        sys.exit(1)
 764
 765    # Format the program (formatter ensures trailing newline)
 766    formatted_text = program_to_text(program, **formatting_options)
 767
 768    # Check if changes are needed
 769    needs_formatting = original_text != formatted_text
 770
 771    # Handle --diff flag
 772    if show_diff:
 773        if needs_formatting:
 774            # Show unified diff
 775            diff = difflib.unified_diff(
 776                original_text.splitlines(keepends=True),
 777                formatted_text.splitlines(keepends=True),
 778                fromfile=f"{file_path} (original)",
 779                tofile=f"{file_path} (formatted)",
 780            )
 781            sys.stdout.writelines(diff)
 782        else:
 783            print(f" {file_path} is already formatted", file=sys.stderr)
 784
 785    # Handle --check mode (or --diff mode - both don't modify files)
 786    if check_mode or show_diff:
 787        if needs_formatting and not show_diff:
 788            # Only print this if we didn't already show diff
 789            print(f" {file_path} would be reformatted", file=sys.stderr)
 790        elif not needs_formatting and not show_diff:
 791            # Only print if we didn't already print in diff section
 792            print(f" {file_path} is already formatted", file=sys.stderr)
 793        return needs_formatting
 794
 795    # Default mode: write formatted output
 796    if needs_formatting:
 797        try:
 798            path.write_text(formatted_text)
 799            print(f" Formatted {file_path}", file=sys.stderr)
 800        except IOError as e:
 801            print(f"Error writing to {file_path}: {e}", file=sys.stderr)
 802            sys.exit(1)
 803    else:
 804        print(f" {file_path} is already formatted", file=sys.stderr)
